# Glossary

Key terms and definitions used throughout the Nousio platform.

---

| Term | Definition |
|------|-----------|
| **Company** | The top-level tenant entity. All data is scoped to a company. Each company has its own users, documents, conversations, and AI context. |
| **Workspace** | Synonym for Project. A named context (e.g., a client or initiative) that scopes documents and task boards. |
| **Project** | A named workspace within a company that organizes documents and task boards. Companies can have multiple projects. |
| **User** | An authenticated person within a company. Has a role (MEMBER, ADMIN, SUPER_ADMIN) and belongs to access groups. |
| **Document** | A file uploaded to the company's knowledge base. Automatically processed for text extraction and embedding generation. |
| **Knowledge Source** | A document marked with `useAsKnowledgeSource: true`. Knowledge sources receive a retrieval boost (+0.15) during RAG. |
| **Context** | The combination of company profile data, RAG-retrieved knowledge chunks, and AI Brain instructions that are injected into AI prompts. |
| **Conversation** | An AI chat session containing an ordered sequence of messages. Scoped to a company and a specific assistant type. |
| **Message** | A single turn in a conversation. Has a role (USER, ASSISTANT, SYSTEM) and text content. |
| **Growth Assistant** | An AI content generation module backed by an `AIBrainProfile` team member. All assistants are custom-created — no predefined types. Each generates content grounded in company knowledge. |
| **Generation Run** | A record of one AI generation request, including the input prompt, output text, context used, and metadata. |
| **Draft** | A saved AI-generated output that the user has chosen to keep. Can be edited and reused. |
| **AI Brain** | A configurable personality and behavior profile for each AI team member. Controls formality, reasoning depth, output style, and source preferences. All brains are custom-created via the AI Team page. |
| **Brain Version** | A snapshot of an AI Brain configuration at a point in time. Supports rollback and audit. |
| **Resource** | A sidebar link to an AI team member. The RESOURCES section is fully dynamic — only `AIBrainProfile` records that exist appear. 1 member = 1 resource. |
| **RAG** | Retrieval-Augmented Generation. The process of finding relevant company knowledge chunks and injecting them into AI prompts to ground responses in real data. |
| **Embedding** | A 1536-dimension numerical vector representation of a text chunk, generated by OpenAI's `text-embedding-3-small` model. Used for semantic similarity search. |
| **Chunk** | A segment of a document's text (~500 words) created during the embedding pipeline. Documents are split into overlapping chunks for better retrieval. |
| **Tenant** | A company in the multi-tenant architecture. All data is isolated per tenant via `companyId` scoping. |
| **Indexing** | The process of generating and storing embeddings for a document's text chunks. Makes the document searchable via RAG. |
| **Retrieval** | The process of finding the most relevant knowledge chunks for a given query using semantic similarity (cosine similarity). |
| **Agent** | An AI Brain configured for a specific assistant type (e.g., SALES, MARKETING). Determines how the AI behaves. |
| **Access Group** | A named group of users with specific feature permissions. Users can belong to multiple groups. |
| **Permission Grant** | A specific permission assigned to an access group: a resource type (FEATURE / SUB_FEATURE / PROJECT_SCOPE), a resource key, and an access level (VIEW / USE / MANAGE). |
| **Effective Access** | The computed union of all permissions from a user's access groups. Determines what the user can see and do. |
| **Feature Key** | A string identifier for a platform feature (e.g., `marketing`, `crm`, `tasks`). Used in permission grants and feature toggles. |
| **Super Admin** | A platform-level administrator who can access the Backoffice and manage all companies. |
| **Backoffice** | The super admin panel for managing companies, users, features, and platform analytics. Separate from the main dashboard. |
| **Onboarding** | The 6-step setup wizard that guides new users through company profile creation, document upload, and initial configuration. |
| **CRM** | Customer Relationship Management. A lightweight pipeline for tracking leads through sales stages. |
| **Pipeline Stage** | A column in the CRM pipeline (e.g., "New Lead", "Qualified", "Proposal", "Won"). Leads move between stages. |
| **Lead** | A potential customer in the CRM. Has contact info, pipeline stage, lifecycle status, and activity history. |
| **Task Board** | A Kanban-style board with columns for organizing tasks. Can be scoped to a project. |
| **Classification** | A custom AI data extraction schema defined by the user. Applied to documents to extract structured fields. |
| **Knowledge Gap** | A topic that AI assistants cannot adequately answer. Auto-detected from ungrounded questions. |
| **Grounding Status** | The degree to which an AI response is supported by company knowledge: VERIFIED, PARTIAL, NOT_FOUND, or GENERAL. |
| **Tactile Brutalism** | The platform's design system. Characterized by 0px border-radius, bold 2px-4px borders, 4px shadow offsets, uppercase typography, and Nousio Blue (#2563eb) accent. |
| **OCR Status** | The processing lifecycle of a document's OCR: `PENDING` → `PROCESSING` → `PROCESSED` → `ERROR`. Stored in `Document.ocrStatus`. |
| **Document Version** | An integer tracking how many times a document's content has been updated. Incremented when a file with the same name but different SHA-256 hash is re-uploaded. |
| **Content Hash** | A SHA-256 hash of a document's file content. Used for duplicate detection and version change detection. |
| **Bulk Action Bar** | A dark-themed UI bar that appears when documents are selected. Provides batch operations: Reprocess, Category, Priority, Curated, Delete. |
| **External Document** | A document synced from Google Drive or Notion. Stored in the `ExternalDocument` table with an `ext-` prefixed ID in the frontend. |
